# 复杂度分析

## 是什么

复杂度分析是整个算法许欸小的而精髓，只要掌握了它，数据结构和算法的内容基本上就掌握了一半

## 为什么学

* 测试结果非常依赖测试环境
* 测试结果手数据规模的影响

## 学什么

### 大 O 复杂度表示法

大 O 时间复杂度表示法实际上并不是表示真正的执行时间，而是表示代码执行时间随数据规模增长的变化趋势，也叫渐进时间复杂度

``` bash
  T(n) = O(f(n))
```

对于公式的低阶、常量、系数三部分并不左右增长趋势，所以可以忽略

### 分析方法

1. 只关注循环次数最多的一段代码
2. 加法法则：总复杂度等于量级最大的那段代码的复杂度
3. 乘法法则：嵌套代码的复杂度等于嵌套内外代码复杂度的乘积
